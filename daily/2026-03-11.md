<details> <summary><strong>1. Valid Palindrome</strong> — Easy · Two Pointers</summary>
🔗 링크: https://leetcode.com/problems/valid-palindrome/

문제 요구사항

영숫자만 남기고 대소문자 무시 후 팰린드롬 판별

내가 작성한 코드 (오답)

javascript
function preprocess(s) {
  return s
    .toLowerCase()
    .replace(/[^a-z0-9]/g, '');
}

function isPalindrome(s) {
    let returnBoolean = false;
    const transferText = preprocess(s);
    if(transferText.length % 2 !== 0) return false;  // ❌ 버그 1
    for(let i = 0; i < transferText.length/2; i++){
        if(transferText[i] === transferText[transferText.length - i -1]) return true;  // ❌ 버그 2
    }
    return returnBoolean;
};
틀린 이유

❌ 버그 1: length % 2 !== 0이면 false 반환 — "racecar"처럼 홀수 길이도 팰린드롬 가능. 이 조건 자체가 잘못됨

❌ 버그 2: 루프 내에서 한 번이라도 같으면 return true — 끝까지 모두 같아야 팰린드롬인데 첫 글자만 같아도 true가 됨

핵심 패턴

Two Pointers — left/right 양끝에서 좁히며 불일치 시 즉시 false

올바른 풀이

javascript
// 방법 1: 전처리 후 Two Pointers
function isPalindrome(s: string): boolean {
    const clean = s.toLowerCase().replace(/[^a-z0-9]/g, '');
    let left = 0, right = clean.length - 1;
    while (left < right) {
        if (clean[left] !== clean[right]) return false;
        left++;
        right--;
    }
    return true;
}

// 방법 2: 전처리 없이 직접 Two Pointers — O(1) 공간
function isPalindrome(s: string): boolean {
    let left = 0, right = s.length - 1;
    const isAlNum = (c: string) => /[a-z0-9]/.test(c.toLowerCase());
    while (left < right) {
        while (left < right && !isAlNum(s[left])) left++;
        while (left < right && !isAlNum(s[right])) right--;
        if (s[left].toLowerCase() !== s[right].toLowerCase()) return false;
        left++; right--;
    }
    return true;
}
💡 방법 2가 추가 문자열 생성 없이 O(1) 공간으로 더 효율적

<details> <summary><strong>2. Two Sum II</strong> — Medium · Two Pointers</summary>
🔗 링크: https://leetcode.com/problems/two-sum-ii-input-array-is-sorted/

문제 요구사항

정렬된 배열에서 합이 target인 두 인덱스 반환 (1-indexed)

내가 작성한 코드 (비효율적 풀이)

javascript
function twoSum(numbers: number[], target: number): number[] {
    const returnArray = [];
    for(let i=0; i<numbers.length; i++){
       for(let j=0; j<numbers.length; j++){
            if(i !==j && target - numbers[i] === numbers[j]){  // ⚠️ O(n²)
                 returnArray.push(i + 1)
                 returnArray.push(j + 1)
                 return returnArray
            }
       }
    }
};
틀린 이유

⚠️ O(n²) 이중 루프 — 동작은 하지만 정렬된 배열의 특성을 전혀 활용하지 않음

⚠️ i !== j 조건만으로는 [i, j]와 [j, i] 중복 반환 가능 (i < j 조건 필요)

⚠️ 면접에서 Two Pointers를 기대하는 문제에 브루트포스는 감점 요소

핵심 패턴

정렬된 배열 + Two Pointers — 합이 크면 right--, 작으면 left++

올바른 풀이

javascript
function twoSum(numbers: number[], target: number): number[] {
    let left = 0, right = numbers.length - 1;
    while (left < right) {
        const sum = numbers[left] + numbers[right];
        if (sum === target) return [left + 1, right + 1];
        else if (sum < target) left++;
        else right--;
    }
    return [];
}
💡 핵심 아이디어: 정렬되어 있으므로 합이 크면 오른쪽을 줄이고, 작으면 왼쪽을 늘리면 O(n)

<details> <summary><strong>3. 3Sum</strong> — Medium · Two Pointers ⚠️ 못 품</summary>
🔗 링크: https://leetcode.com/problems/3sum/

문제 요구사항

합이 0인 세 수 조합 반환, 중복 없이

못 푼 이유

Two Pointers 패턴 자체가 처음이라 정상 — 처음에는 보고 패턴 익히는 것이 목표

핵심 패턴

정렬 후 i 고정 → Two Pointers로 나머지 두 수 탐색 + 중복 스킵

올바른 풀이

javascript
function threeSum(nums: number[]): number[][] {
    nums.sort((a, b) => a - b);
    const result: number[][] = [];

    for (let i = 0; i < nums.length - 2; i++) {
        if (i > 0 && nums[i] === nums[i - 1]) continue;  // 중복 스킵
        let left = i + 1, right = nums.length - 1;
        while (left < right) {
            const sum = nums[i] + nums[left] + nums[right];
            if (sum === 0) {
                result.push([nums[i], nums[left], nums[right]]);
                while (left < right && nums[left] === nums[left + 1]) left++;
                while (left < right && nums[right] === nums[right - 1]) right--;
                left++; right--;
            } else if (sum < 0) left++;
            else right--;
        }
    }
    return result;
}
💡 핵심 아이디어: i를 고정하면 "합이 -nums[i]인 Two Sum" 문제로 변환됨
중복 스킵 조건 if (i > 0 && nums[i] === nums[i-1]) continue 가 핵심

function maxArea(height: number[]): number {
    let left = 0, right = height.length - 1;
    let maxWater = 0;
    while (left < right) {
        const water = Math.min(height[left], height[right]) * (right - left);
        maxWater = Math.max(maxWater, water);
        if (height[left] < height[right]) left++;
        else right--;
    }
    return maxWater;
}
💡 왜 짧은 쪽을 이동하나? 긴 쪽을 이동하면 가로(거리)가 줄고 높이도 줄거나 같으므로 넓이가 절대 커질 수 없기 때문
