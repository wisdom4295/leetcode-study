<summary><strong>1. Two Sum</strong> — Easy · Array, HashMap · ⏱️ 25분 · 해설참고: N</summary>

<br>

- 🔗 링크: https://leetcode.com/problems/two-sum/

**문제 요구사항**
- 정수 배열 `nums`와 목표값 `target`이 주어질 때 두 수의 합이 `target`이 되는 두 인덱스 반환
- 같은 인덱스를 두 번 사용할 수 없음, 정확히 하나의 답이 존재한다고 가정

**핵심 패턴**
> 인덱스 저장 HashMap, 보완값(target - nums[i]) 탐색

**내 풀이 코드**
```javascript
function twoSum(nums: number[], target: number): number[] {
    const returnArray = [];
    for(let i=0; i<nums.length; i++){
       for(let j=0; j<nums.length; j++){
            if(i !==j && target-nums[i] === nums[j]){
                 returnArray.push(i)
                 returnArray.push(j)
                 return returnArray
            }
       }
    }
};
```

**다른 풀이 방법**
```javascript
function twoSum(nums: number[], target: number): number[] {
    let m = new Map<number,number>
    for(let i = 0; i<nums.length; i++){
        const c = target - nums[i]
        if(m.has(c)) return [m.get(c),i]
        m.set(nums[i],i)
    }
};
```

> 💡 내 풀이와의 차이점:
> - 내 풀이는 이중 for문으로 **O(n²)** 시간복잡도, 다른 풀이는 Map 한 번 순회로 **O(n)**
> - 다른 풀이는 `m.set(nums[i], i)`로 값→인덱스를 저장해두고, 다음 순회에서 보완값이 이미 Map에 있는지 `m.has(c)`로 즉시 확인
> - 핵심 아이디어: **"지나온 숫자를 Map에 기록하면서 한 번만 순회"**

**잘한 점**
- 이중 for문으로 브루트포스 접근을 스스로 구현한 것 자체는 논리적으로 정확함
- `i !== j` 조건으로 같은 인덱스 사용 금지 엣지케이스를 놓치지 않음
- 문제 의도(인덱스 2개 반환)를 정확히 파악함

**잘못한 점 / 개선할 점**
- 이중 for문 → **O(n²)** 으로 배열이 길어질수록 성능 급격히 저하
- Map을 활용한 **O(n)** 단일 순회 패턴을 익혀야 함
- `target - nums[i]` 라는 보완값 개념을 먼저 떠올리는 훈련 필요

<summary><strong>2. Best Time to Buy and Sell Stock</strong> — Easy · Array · ⏱️ 25분 · 해설참고: N</summary>

<br>

- 🔗 링크: https://leetcode.com/problems/best-time-to-buy-and-sell-stock/

**문제 요구사항**
- 주가 배열 `prices`가 주어질 때 한 번만 사고팔 수 있을 때 최대 이익 반환
- 이익을 낼 수 없으면 0 반환

**핵심 패턴**
> 최솟값 갱신하며 최대 이익 계산 (Greedy)

**내 풀이 코드**
```javascript
function maxProfit(prices: number[]): number {
    let returnBenefit = 0;
    for(let i=0; i < prices.length; i++){
        for(let j=1; j < prices.length; j++){
            if(prices[j] - prices[i] >= returnBenefit){
                returnBenefit = prices[j] - prices[i]
                return returnBenefit  // ⚠️ 첫 양수 차이에서 바로 리턴되는 버그
            }
        }
    }
};
```

**다른 풀이 방법**
```javascript
function maxProfit(prices: number[]): number {
    let minPrice = prices;
    let maxProfit = 0;
    for(let i = 1; i < prices.length; i++){
        if(prices[i] < minPrice){
            minPrice = prices[i];
        } else if(prices[i] - minPrice > maxProfit){
            maxProfit = prices[i] - minPrice;
        }
    }
    return maxProfit;
};
```

> 💡 내 풀이와의 차이점:
> - 내 풀이는 `return`을 루프 안에 넣어서 **첫 번째 양수 이익이 나오는 순간 종료** → 최대 이익이 아닌 첫 이익을 반환하는 버그 존재
> - 다른 풀이는 `minPrice`를 갱신하면서 단일 순회로 **O(n)** 처리
> - 핵심 아이디어: **"지금까지의 최솟값을 추적하고, 현재값 - 최솟값이 최대이익보다 크면 갱신"**

**잘한 점**
- `prices[j] - prices[i]`로 이익 계산 방향(사는 날 < 파는 날)은 정확히 이해함
- 초기값을 0으로 설정해서 이익이 없을 때 0 반환하는 조건을 자연스럽게 처리함

**잘못한 점 / 개선할 점**
- 루프 안에서 `return`을 사용하면 **최대값 탐색이 아닌 조건 만족 즉시 종료**가 됨 — 최댓값을 구할 때는 루프를 다 돌고 마지막에 `return`
- 이중 for문 대신 `minPrice` 변수 하나로 단일 순회하는 Greedy 패턴 익히기
- `Math.max(...prices) - Math.min(...prices)` 접근은 날짜 순서(사기 전에 팔 수 없음)를 무시하므로 틀린 로직임을 인지

</details>

<summary><strong>3. Contains Duplicate</strong> — Easy · Array, Hash Table · ⏱️ 25분 · 해설참고: N</summary>

<br>

- 🔗 링크: https://leetcode.com/problems/contains-duplicate/

**문제 요구사항**
- 정수 배열 `nums`에서 같은 값이 두 번 이상 등장하면 `true` 반환
- 모든 값이 다르면 `false` 반환

**핵심 패턴**
> Set으로 중복 체크 (Set.size vs array.length 비교)

**내 풀이 코드**
```javascript
function containsDuplicate(nums: number[]): boolean {
    let returnBoolean = false;
    for(let i=0; i < nums.length; i++){
        for(let j=1; j < nums.length; j++){
            if(i !== j && nums[i] === nums[j]){
                return returnBoolean = true
            }
            return returnBoolean  // ⚠️ 첫 번째 j 순회 후 바로 리턴되는 버그
        }
    }
};
```

**다른 풀이 방법**
```javascript
function containsDuplicate(nums: number[]): boolean {
    return new Set(nums).size !== nums.length;
};
```

> 💡 내 풀이와의 차이점:
> - 내 풀이는 `return returnBoolean`이 내부 for문 안에 있어서 **j=1 한 번만 비교하고 항상 종료** → 대부분의 케이스에서 잘못된 결과 반환
> - 다른 풀이는 `Set`에 배열을 넣으면 중복이 자동 제거되므로 **원본 길이와 Set 크기가 다르면 중복 존재**로 한 줄 처리
> - 핵심 아이디어: **`new Set(nums).size !== nums.length`**

**잘한 점**
- `i !== j` 조건으로 같은 인덱스 비교 방지 로직은 올바르게 생각함
- 중복 비교라는 문제 본질을 정확히 파악함

**잘못한 점 / 개선할 점**
- `return`의 위치가 if 블록 밖(내부 for문 안)에 있어서 **무조건 첫 루프에서 탈출**하는 버그 — 들여쓰기와 return 위치를 항상 재확인할 것
- Set 자료구조의 특성(중복 불허)을 활용하면 훨씬 간결하게 해결 가능

</details>

<summary><strong>4. Valid Anagram</strong> — Easy · String, Hash Table · ⏱️ 25분 · 해설참고: N</summary>

<br>

- 🔗 링크: https://leetcode.com/problems/valid-anagram/

**문제 요구사항**
- 두 문자열 `s`, `t`가 주어질 때 `t`가 `s`의 애너그램이면 `true` 반환
- 길이가 다르면 즉시 `false`

**핵심 패턴**
> 문자 빈도수 HashMap 비교 (s는 +1, t는 -1 후 0 확인)

**내 풀이 코드**
```javascript
function charCount(str) {
  const obj = {};
  for (const char of str) {
    obj[char] = (obj[char] ?? 0) + 1;
  }
  return obj;
}

function isAnagram(s: string, t: string): boolean {
    let returnBoolean = false;
    if(JSON.stringify(charCount(s), Object.keys(charCount(s)).sort()) ===
       JSON.stringify(charCount(t), Object.keys(charCount(t)).sort())) returnBoolean = true;
    return returnBoolean
};
```

**다른 풀이 방법**
```javascript
function isAnagram(s: string, t: string): boolean {
    // 방법 1: 정렬 비교 (간결)
    // return s.split('').sort().join('') === t.split('').sort().join('');

    // 방법 2: charCode 배열 (최적화)
    let sc = Array(26).fill(0);
    let tc = Array(26).fill(0);
    for (let i = 0; i < s.length; ++i) ++sc[s.charCodeAt(i) - 97];
    for (let i = 0; i < t.length; ++i) ++tc[t.charCodeAt(i) - 97];
    return sc.join() === tc.join();
};
```

> 💡 내 풀이와의 차이점:
> - 내 풀이는 `JSON.stringify`로 객체를 문자열 비교하는 창의적 접근이나, **키 순서 보장이 불안정**해서 엣지케이스에서 틀릴 수 있음
> - 방법 1(`sort().join()`)이 가장 직관적이고 안전한 정렬 비교
> - 방법 2(`charCodeAt - 97`)는 알파벳 소문자 26개를 인덱스로 매핑해서 **O(n)** 으로 가장 빠름
> - 핵심 아이디어: **`charCodeAt(i) - 97`로 'a'=0, 'b'=1 ... 'z'=25 인덱스 변환**

**잘한 점**
- `charCount` 헬퍼 함수로 분리해서 재사용 가능하게 설계한 점이 좋음
- `obj[char] ?? 0` nullish coalescing 활용이 깔끔함
- 문자 빈도수 비교라는 핵심 접근 방향은 정확함

**잘못한 점 / 개선할 점**
- `JSON.stringify` 두 번째 인자는 replacer 함수/배열이지, 키 정렬 옵션이 아님 → 의도대로 동작하지 않음
- 객체 키 순서는 JS 엔진마다 다를 수 있어 `JSON.stringify` 직접 비교는 신뢰하기 어려움
- 길이가 다르면 즉시 `false` 반환하는 **early return** 추가하면 성능 개선 가능: `if(s.length !== t.length) return false`

</details>
