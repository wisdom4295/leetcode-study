<summary><strong>1. Group Anagrams</strong> — Medium · Array, HashMap</summary>

<br>

- 🔗 링크: https://leetcode.com/problems/group-anagrams/

**문제 요구사항**
- 문자열 배열 `strs`가 주어질 때 애너그램끼리 묶어서 2차원 배열로 반환
- 순서는 상관없음

**핵심 패턴**
> 정렬된 문자열을 HashMap의 key로 사용해 그룹핑

**막힌 부분**
-

**다음에 시도할 접근법**
- 각 단어를 `split('').sort().join('')` 으로 정렬 → 같은 key끼리 Map에 묶기

**참고할 풀이 방법**
```javascript
// 방법 1: 정렬 key HashMap (직관적)
function groupAnagrams(strs: string[]): string[][] {
    const map = new Map<string, string[]>();
    for (const s of strs) {
        const key = s.split('').sort().join('');
        if (!map.has(key)) map.set(key, []);
        map.get(key)!.push(s);
    }
    return Array.from(map.values());
}

// 방법 2: charCode 빈도수 key — O(n * m) 최적화 (정렬 없이)
function groupAnagrams(strs: string[]): string[][] {
    const map = new Map<string, string[]>();
    for (const s of strs) {
        const count = Array(26).fill(0);
        for (const c of s) count[c.charCodeAt(0) - 97]++;
        const key = count.join('#');
        if (!map.has(key)) map.set(key, []);
        map.get(key)!.push(s);
    }
    return Array.from(map.values());
}
```

> 💡 두 풀이의 차이점:
> - 방법 1은 각 단어를 정렬해서 key로 사용 → **O(n * m log m)** (m: 단어 길이)
> - 방법 2는 정렬 없이 charCode 빈도수 배열을 key로 사용 → **O(n * m)** 으로 더 빠름
> - 핵심 아이디어: **애너그램은 문자 구성이 같으므로 정렬하면 항상 같은 문자열이 됨**

<summary><strong>2. Top K Frequent Elements</strong> — Medium · Array, HashMap</summary>

<br>

- 🔗 링크: https://leetcode.com/problems/top-k-frequent-elements/

**문제 요구사항**
- 정수 배열 `nums`와 정수 `k`가 주어질 때 빈도수 상위 k개의 요소 반환
- 시간복잡도 O(n log n) 이하로 풀기

**핵심 패턴**
> 빈도수 Map 생성 후 빈도 기준 내림차순 정렬, 상위 k개 슬라이싱

**막힌 부분**
-

**다음에 시도할 접근법**
- `Map`으로 빈도수 기록 → `Array.from(map.entries()).sort((a,b) => b[1]-a[1])` → 앞에서 k개 추출

**참고할 풀이 방법**
```javascript
// 방법 1: Map + sort (직관적)
function topKFrequent(nums: number[], k: number): number[] {
    const map = new Map<number, number>();
    for (const n of nums) {
        map.set(n, (map.get(n) ?? 0) + 1);
    }
    return Array.from(map.entries())
        .sort((a, b) => b - a)[1]
        .slice(0, k)
        .map(([val]) => val);
}

// 방법 2: Bucket Sort — O(n) 최적화
function topKFrequent(nums: number[], k: number): number[] {
    const map = new Map<number, number>();
    for (const n of nums) {
        map.set(n, (map.get(n) ?? 0) + 1);
    }

    const bucket: number[][] = Array.from({ length: nums.length + 1 }, () => []);
    for (const [val, freq] of map.entries()) {
        bucket[freq].push(val);
    }

    const result: number[] = [];
    for (let i = bucket.length - 1; i >= 0 && result.length < k; i--) {
        result.push(...bucket[i]);
    }
    return result;
}
```

> 💡 두 풀이의 차이점:
> - 방법 1은 `sort`를 사용해 **O(n log n)**, 구현이 직관적
> - 방법 2는 빈도수를 인덱스로 하는 **버킷 배열**을 활용해 **O(n)** — 빈도수는 최대 `nums.length`를 넘을 수 없다는 점을 이용

<summary><strong>3. Product of Array Except Self</strong> — Medium · Array</summary>

<br>

- 🔗 링크: https://leetcode.com/problems/product-of-array-except-self/

**문제 요구사항**
- 정수 배열 `nums`에서 자기 자신을 제외한 나머지 원소의 곱을 담은 배열 반환
- 나눗셈 사용 금지, 시간복잡도 O(n)

**핵심 패턴**
> 왼쪽 누적곱 배열 + 오른쪽 누적곱 배열을 각각 만들어 곱함 (양방향 순회)

**막힌 부분**
-

**다음에 시도할 접근법**
- prefix 배열: `prefix[i] = prefix[i-1] * nums[i-1]`
- suffix 배열: `suffix[i] = suffix[i+1] * nums[i+1]`
- 결과: `prefix[i] * suffix[i]`

**참고할 풀이 방법**
```javascript
// 방법 1: prefix / suffix 배열 분리 (직관적)
function productExceptSelf(nums: number[]): number[] {
    const n = nums.length;
    const prefix = Array(n).fill(1);
    const suffix = Array(n).fill(1);

    for (let i = 1; i < n; i++) {
        prefix[i] = prefix[i - 1] * nums[i - 1];
    }
    for (let i = n - 2; i >= 0; i--) {
        suffix[i] = suffix[i + 1] * nums[i + 1];
    }

    return prefix.map((val, i) => val * suffix[i]);
}

// 방법 2: 공간 최적화 — O(1) extra space
function productExceptSelf(nums: number[]): number[] {
    const n = nums.length;
    const result = Array(n).fill(1);

    let prefix = 1;
    for (let i = 0; i < n; i++) {
        result[i] = prefix;
        prefix *= nums[i];
    }

    let suffix = 1;
    for (let i = n - 1; i >= 0; i--) {
        result[i] *= suffix;
        suffix *= nums[i];
    }

    return result;
}
```

> 💡 두 풀이의 차이점:
> - 방법 1은 prefix/suffix 배열을 각각 만들어 **O(n) 공간** 사용, 흐름이 명확해 이해하기 쉬움
> - 방법 2는 result 배열 하나에 prefix를 먼저 채운 뒤 suffix를 뒤에서 곱해 **O(1) 추가 공간**으로 최적화
> - 핵심 아이디어: **i번째 결과 = i 왼쪽 누적곱 × i 오른쪽 누적곱**

<summary><strong>4. Longest Consecutive Sequence</strong> — Medium · Array, Set</summary>

<br>

- 🔗 링크: https://leetcode.com/problems/longest-consecutive-sequence/

**문제 요구사항**
- 정렬되지 않은 정수 배열 `nums`에서 연속된 숫자 시퀀스의 최대 길이 반환
- 시간복잡도 O(n)

**핵심 패턴**
> Set으로 변환 후 `n-1`이 Set에 없는 값만 시작점으로 삼아 `n+1`, `n+2`... 연속 탐색

**막힌 부분**
-

**다음에 시도할 접근법**
- `const s = new Set(nums)`
- `s.has(n-1)`이 false인 경우만 시작점으로 설정
- `while(s.has(current + 1))` 로 연속 길이 카운팅

**참고할 풀이 방법**
```javascript
// 방법 1: Set 시작점 탐색 — O(n)
function longestConsecutive(nums: number[]): number {
    const s = new Set(nums);
    let longest = 0;

    for (const n of s) {
        if (!s.has(n - 1)) {  // 시작점인 경우만
            let length = 1;
            while (s.has(n + length)) length++;
            longest = Math.max(longest, length);
        }
    }
    return longest;
